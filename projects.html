Banking-App Prototype

Source Code

The Banking Application is a Python-based software solution that emulates a banking system, offering functionalities such as account creation, login, and various banking operations like balance checking, withdrawal, depositing, and fund transfers.

Features:

  • Secure Login: Allows users to log in with their username and PIN to access their accounts.
  • Account Creation: Provides a feature to create a new account with a unique username and PIN.
  • Balance Check: Enables users to check their account balance at any time.
  • Withdrawal & Deposit: Facilitates the withdrawal and deposit of funds into user accounts.
  • Transfer Funds: Allows users to transfer money between accounts securely.

Implementation Details:

  • Modular Structure: Organized into different classes and functions for specific functionalities, ensuring code readability and maintainability.
  • Data Management: Stores user details (username, balance, name, PIN) in a CSV file (names2.csv).
  • Security Measures: Includes user input validation and implements encryption techniques to secure sensitive information like PINs.
  • User Interface: Utilizes Tkinter, a Python GUI toolkit, to create a user-friendly interface for interacting with the banking system.

Technologies Used:

  • Python: Employed Python for the backend logic and functionality of the banking operations.
  • Tkinter: Utilized Tkinter for creating the graphical user interface elements and user interaction.
  • CSV Data Handling: Implemented CSV file management for storing and retrieving user data.
  • Security Measures: Utilized encryption techniques for securing sensitive user information.

The Banking Application demonstrates my proficiency in software development, user interface design, and implementing secure banking functionalities. It ensures a user-friendly experience while prioritizing security measures to protect user data and transactions.

Color-Following Game

Source Code

The Color-Following Game is a Python-based interactive challenge that tests a player's memory and responsiveness by flashing colors in a sequence that the player needs to replicate. Developed using a custom GUI library, the game provides an engaging platform for users to follow and recreate complex color patterns.

Features:

  • Memory Challenge: Players must replicate a sequence of colors shown by clicking corresponding buttons.
  • Difficulty Levels: Offers multiple difficulty modes, each with faster sequences and shorter time to respond.
  • High Score Tracking: Keeps track of the highest scores achieved in different difficulty levels.
  • Interactive Interface: Uses vibrant color flashes and sound effects to engage players.

Implementation Details:

  • Sequence Generation: Randomly generates color sequences of varying lengths based on the selected difficulty level.
  • User Response Check: Validates player input against the displayed color sequence.
  • Sound Effects: Utilizes sound effects to enhance the gaming experience.
  • Score Tracking: Keeps track of the current and highest scores achieved by players.

Technologies Used:

  • Python: Employed Python for game logic and functionality.
  • Custom GUI Library: Developed using a custom GUI library (Customtkinter) for creating the graphical user interface.
  • Randomization: Utilized Python's random module for generating random sequences.
  • Pygame: Integrated Pygame for sound effects and audio playback.

The Color-Following Game showcases my proficiency in game development, user interface design, and logic implementation. It demonstrates an engaging and challenging gaming experience while assessing players' memory and responsiveness.

Connect Four Game

Source Code

The Connect Four game is a Python-based rendition of the classic tabletop game where two players take turns dropping colored discs into a vertical grid. The objective is to form a line of four discs of the same color, either vertically, horizontally, or diagonally, to win the game.

Features:

  • Interactive Gameplay: Players take turns dropping discs into the grid by clicking on the desired column.
  • Winning Condition: Detects and announces a win when a player successfully forms a line of four discs.
  • Dynamic Grid: Utilizes a 7x7 grid to provide a playing field for the Connect Four game.
  • Color-Coded Discs: Represents player moves with distinct colors for easy identification.

Implementation Details:

  • Grid Representation: Implements a grid structure to manage player moves and track the game state.
  • Win Detection Algorithm: Checks for winning combinations horizontally, vertically, and diagonally after each move.
  • User Interface: Uses a custom GUI library to create an interactive and visually appealing game board.

Technologies Used:

  • Python: Utilized Python programming for game logic and GUI development.
  • Custom GUI Library: Developed using a custom GUI library (Customtkinter) for the game's graphical user interface.
  • PIL (Python Imaging Library): Integrated PIL for image handling and displaying player discs.

The Connect Four game demonstrates my proficiency in game development, logic implementation, and graphical user interface design using Python. It offers an enjoyable gaming experience while challenging players' strategic thinking and pattern recognition skills.

Features:

  • Graphics Handling: Incorporated image handling with PhotoImage to integrate a clock image into the interface.

Technologies Used:

  • Python: Utilized Python programming language for the backend logic.
  • Tkinter: Leveraged Tkinter library for creating the graphical user interface.
  • Time Module: Utilized the strftime function from the time module to fetch system time.
